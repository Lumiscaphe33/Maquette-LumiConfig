Questions ouvertes — à trancher avant développement

Les 5 décisions critiques identifiées par le rapport d'audit. Sans réponse à ces questions, le modèle 585 €/mois s'effondre.

Q1 · Faisabilité technique
Patchwork 3D existe-t-il en mode CLI headless containerisable (Linux ARM) ?
Impact : si NON, le pipeline cloud auto échoue → soit dev d'un mode headless (gros chantier), soit pivot sur SDK tiers (InstaLOD ~165$/mois/poste, CAD Exchanger ~2 690$ + volume). Bloque toute la promesse 15 min.
Q2 · UV unwrap automatisé
Le moteur P3D peut-il faire de l'UV unwrap tri-planaire 100% auto sur des assemblages CAO bruts ?
Impact : l'étape la plus chronophage / frustrante de la 3D. Si pas auto → utilisateur doit intervenir → promesse TTV cassée. Validation pratique sur un cas réel VDL exigée avant Gate 2.
Q3 · AxF accessible API
La bibliothèque AxF Lumiscaphe est-elle exposable via API + filtrable par tag sectoriel ?
Impact : l'AxF est le différenciateur face à KeyShot/Threekit. Si la lib n'est pas filtrable → fatigue décisionnelle → adoption marketing manager bloquée. Catalogue à structurer côté backend.
Q4 · Pricing usage-based
L'infra peut-elle mesurer fiablement les "vues" (1 chargement viewer = 1 vue ?) pour facturer au-delà du quota ?
Impact : sans cette mesure, l'Egress AWS peut bouffer 30% de la marge sur un client trafiqué (rapport audit, 180€/mo). Nécessite tracking + alerte + UI conso côté client (cf. dashboard).
Q5 · Compression obligatoire
Le pipeline export final intègre-t-il Draco (géométrie) + KTX2 (textures) systématiquement ?
Impact : sans cela, poids des configurateurs ≥ 20 MB. Egress ×7 → marges détruites. Compression non-négociable à intégrer dès le build pipeline (étape 4).
Q6 · Caméras vrObject auto-générées
Sur ingestion, génère-t-on automatiquement une caméra vrObject (turntable 24-36 frames) en plus des bookmarks fixes ?
Impact : la vue 360° interactive est attendue par défaut sur configurateurs B2C/B2B 2026. Si pas généré auto → utilisateur doit créer manuellement → friction. Coût pré-render = 36 snapshots WebRender supplémentaires.
